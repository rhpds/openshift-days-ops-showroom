= OpenShift Days - Ops Track

== Why This Workshop Exists

Running Kubernetes in production is hard. Running it well — with enterprise authentication, automated backup, fleet management, and security compliance — is where most teams struggle.

OpenShift closes that gap. It takes upstream Kubernetes and adds the operational tooling that platform teams actually need: integrated monitoring, automated upgrades, built-in security policies, multi-cluster management, and developer self-service — all supported by Red Hat.

This workshop puts you in the driver's seat of a live OpenShift cluster so you can see how these capabilities work in practice, not just in slides.

== What You'll Do

Every module is hands-on. You'll deploy real applications, configure real security policies, break things on purpose and fix them, and explore the tools that make Day 2 operations manageable at scale.

== Your Environment

You have full cluster admin access to a production-grade OpenShift cluster with all the operational tooling pre-installed and ready to explore.

=== Workshop Interface

Your workshop runs in a split-panel interface:

* **Left panel** — Workshop instructions (what you're reading now)
* **Right panel** — Tabbed workspace with direct access to your cluster tools
* **Draggable divider** — Drag the border between panels to resize them

=== Tabs

Along the top of the right panel you'll find tabs for each of the tools deployed in your environment:

[cols="1,3"]
|===
|Tab |What it is

|**OCP Console**
|The OpenShift web console — your primary GUI for managing the cluster. Log in with the credentials provided.

|**Terminal**
|A web-based terminal pre-connected to your cluster's bastion host. Run `oc` commands directly — no local setup needed.

|**RHACS**
|Red Hat Advanced Cluster Security — vulnerability scanning, compliance, policy enforcement, and runtime security.

|**Quay**
|Red Hat Quay — enterprise container registry with image scanning and access control.

|**ArgoCD**
|OpenShift GitOps — continuous delivery and application lifecycle management using GitOps principles.

|**Developer Hub**
|Red Hat Developer Hub — a developer portal with software catalogs, golden paths, and self-service templates.
|===

TIP: Switch between tabs as you work through the modules. Some exercises use the terminal, others use the web console or a specific tool's UI.

=== Clickable Commands

Throughout this workshop, you'll see highlighted command blocks like the one below. **Click on them** to automatically switch to the Terminal tab and execute the command.

=== Verify Access

Click the commands below to confirm you're connected:

[source,bash,role="execute"]
----
oc whoami
----

You should see your admin user.

[source,bash,role="execute"]
----
oc get nodes
----

You should see your cluster nodes in `Ready` status.

[source,bash,role="execute"]
----
oc get clusterversion
----

This confirms your cluster version and update channel.
