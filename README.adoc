= OpenShift Days - Ops Track Workshop

Showroom content for the OpenShift Days Ops Track workshop, covering Day 2 operations for OpenShift administrators.

== Workshop Overview

A comprehensive modular workshop for OpenShift administrators. Modules are selectable at catalog order time so instructors can tailor the content to their audience.

* *link:content/modules/ROOT/pages/index.adoc[Setup & Overview]* (15-20 min) - Lab access, Showroom interface, CLI verification, platform overview, console, monitoring, registry, routing, upgrades, operator ecosystem

*Core Operations*

== Module Selection

Modules are selectable at catalog order time. The navigation dynamically shows/hides content based on which modules are enabled.

Catalog parameters (`module_enable_*`) control both:

* Which operators/workloads get deployed
* Which content appears in the workshop navigation

See link:content/modules/ROOT/nav.adoc[nav.adoc] for the conditional navigation logic using `ifeval`.

== Local Development

=== Using local-build scripts (Recommended)

From the repo root, build and preview the workshop with Podman:

[source,sh]
----
./local-build/up.sh
----

Open http://localhost:8080/index.html in your browser.

Other commands:

[cols="1,2"]
|===
| `./local-build/build.sh` | Generate the static site into `./www/` using `site.yml`
| `./local-build/serve.sh` | Serve `./www/` with httpd in Podman (port 8080)
| `./local-build/stop.sh` | Stop the local httpd container
| `./local-build/reset.sh` | Stop, rebuild, and serve again
| `./local-build/status.sh` | Show container status
|===

Optional environment variables (see `local-build/_common.sh`):

* `SHOWROOM_PORT` — host port (default `8080`)
* `SHOWROOM_PULL=1` — pull the httpd image before starting
* `SHOWROOM_SITE_YML` — alternate Antora playbook (default `site.yml`)

=== Using Antora CLI directly

[source,sh]
----
npm i -g @antora/cli@3.1 @antora/site-generator@3.1
antora generate site.yml --stacktrace
----

Output is generated in `./www/`. Serve with `./local-build/serve.sh` or any static file server.

== Theme

This workshop uses a custom theme with click-to-execute functionality:

* Theme repo: https://github.com/jnewsome97/rhdp_showroom_theme_execute
* Release: `rh-one-2025-execute-v3`

The execute button allows code blocks with `role="execute"` to send commands directly to the wetty terminal via `socket.emit`.
